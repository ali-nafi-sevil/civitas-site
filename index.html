VI.

Why this works.

Eight kinds of failure — covered.

  • Functional, negative, session, permission, usability, search, responsive, security.
  • The taxonomy a senior QA engineer would build, encoded as specialised skills.
  • Each skill is loaded per stage and unloaded — context never drifts.

"Generic agents write generic tests. We wrote thirteen specialised skills."

Every run ships a PR.

  • Onboarding, coverage expansion, bug discovery — the artefact is always a pull request you can read, review, and merge.
  • Nothing lives in our cloud. Tests live in your repo, in your language, in your CI.
  • Engineers will actually read them. Engineers will actually edit them.

"We don't comment on your PR. We send you one."

The methodology is the moat.

  • 13 persistent skills. A page repository that survives runs. Stage isolation so context never drifts.
  • A verification gate so every bug claim ships with screenshot evidence.
  • The model is hot-swappable. When the frontier model changes, you get an upgrade, not a rewrite.

"The trick isn't the AI. It's everything around it."
